# ReviewBridge

A local CLI daemon that monitors GitHub PRs and GitLab MRs for review comments from any reviewer, triages them with Claude, and routes approved fixes directly into the right Claude Code session — without breaking your current flow.

## The Problem

When using Claude Code, you hit a constant copy-paste loop:

1. Open a PR or MR
2. Reviewer (human, Copilot, CodeRabbit, or any bot) leaves comments
3. You manually read them on GitHub/GitLab
4. Copy them into Claude Code
5. Claude Code fixes them
6. Back to GitHub/GitLab to verify

ReviewBridge eliminates steps 3 and 4. Comments flow automatically into the right Claude Code session, with a triage step so you stay in control.

## How It Works

```
GitHub PR / GitLab MR
  Review comments (from anyone)
        │
        ▼
ReviewBridge daemon (running locally)
  Silently triages with Claude API
        │
        ▼
Terminal notification + triage dialog
  ✅ 2 fix  ⚠️ 1 your-call  ❌ 1 skip
        │
   You approve
        │
        ▼
claude --resume <session-id> (your existing session, context preserved)
  Fixes applied, commit created
```

## Features

- **Platform agnostic** — GitHub and GitLab (including self-hosted)
- **Reviewer agnostic** — humans, Copilot, CodeRabbit, any bot
- **Context preserved** — always resumes the correct existing Claude Code session
- **Triage first** — Claude pre-evaluates every comment before you see a dialog
- **Non-intrusive** — working on issue C? Comments for issue A get handled in a background session
- **Offline-safe** — catches up on all missed comments when daemon restarts
- **You stay in control** — nothing is fixed without your explicit approval

## Installation

```bash
brew install reviewbridge
```

Or download the binary for your platform from [Releases](https://github.com/ahmedennaime/reviewbridge/releases).

## Quick Start

```bash
# First-time setup
reviewbridge init

# Start the daemon
reviewbridge start

# Check status
reviewbridge status

# View pending comments
reviewbridge queue

# Stop the daemon
reviewbridge stop
```

## Requirements

- [Claude Code](https://claude.ai/code) installed (`claude` on your PATH)
- Anthropic API key
- GitHub token (for GitHub PR comments)
- GitLab token (for GitLab MR comments, optional)

## Configuration

Config lives at `~/.reviewbridge/config.yaml`. Generated by `reviewbridge init`.

```yaml
anthropic_api_key: sk-ant-...

platforms:
  github:
    token: ghp_...
    polling_interval: 60s
  gitlab:
    token: glpat-...
    url: https://gitlab.com        # or your self-hosted URL
    polling_interval: 60s

triage:
  auto_skip_style_comments: true

notifications:
  desktop: true
  terminal: true
```

## Commands

| Command | Description |
|---|---|
| `reviewbridge init` | Guided first-time setup |
| `reviewbridge start` | Start the background daemon |
| `reviewbridge stop` | Stop the daemon |
| `reviewbridge status` | Show tracked sessions and linked PRs/MRs |
| `reviewbridge queue` | View all pending/parked comments |
| `reviewbridge link` | Manually link a session to a branch or PR |
| `reviewbridge install-skill` | Install the `/check-reviews` Claude Code slash command |
| `reviewbridge update` | Check for a newer version |

## Claude Code Slash Command

After running `reviewbridge install-skill`, you can type `/check-reviews` inside any Claude Code session to manually pull pending comments for the current branch.

## Development

### Prerequisites

- Go 1.26.3+
- Docker (for integration and E2E tests)

### Setup

```bash
git clone https://github.com/ahmedennaime/reviewbridge
cd reviewbridge
cp .env.example .env    # fill in your keys
```

### Running

```bash
# Build and run locally
make build
./bin/reviewbridge start

# Or run inside Docker
make dev
```

### Testing

```bash
# Unit tests (no Docker needed)
make test

# Integration tests (uses Docker WireMock — no real credentials needed)
make test-integration

# End-to-end tests
make test-e2e

# Coverage report
make test-cover
```
